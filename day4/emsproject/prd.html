Product Requirements Document (PRD)

Employee Management System (emsproject)

Date: July 2026 | Status: MVP Completed

1. Overview

The Employee Management System (EMS) is a full-stack web application designed to help administrative users securely manage their workforce. It provides a simple, modern interface to view, add, update, and delete employee records.

2. Goals & Objectives

  • Provide a secure, authenticated environment (JWT) for system administrators.
  • Ensure a smooth and responsive user experience using modern React (Vite).
  • Maintain a robust and scalable backend using Java Spring Boot and MySQL.
  • Provide built-in API documentation for easy future integration and developer hand-off.

3. Tech Stack

Component Technology Details
Frontend React (Vite) Vanilla CSS, Axios, React-Router-DOM
Backend Java Spring Boot 3 Spring Security, Spring Data JPA, JWT (io.jsonwebtoken)
Database MySQL 8.x Hosted remotely (72.60.219.201). Tables: `users`, `employee`
API Docs OpenAPI / Swagger springdoc-openapi-starter-webmvc-ui

4. Core Features & Requirements

4.1 Authentication & Authorization

  • Registration: Users can create an administrator account securely. Passwords must be hashed using BCrypt before storing.
  • Login: Users authenticate via Username and Password to receive a stateless JSON Web Token (JWT).
  • Session Management: The JWT must be attached to the Authorization: Bearer header for all subsequent API calls. Unauthenticated requests must be rejected (403/401).

4.2 Employee CRUD Operations

  • Create: Authenticated users can add new employees providing Name, Department, and Salary.
  • Read: Authenticated users can view a tabular list of all employees currently in the database.
  • Update: Users can select an existing employee and edit their Name, Department, or Salary.
  • Delete: Users can completely remove an employee record from the system (soft-delete not required for MVP).

4.3 API Documentation

  • The backend must automatically generate OpenAPI documentation.
  • The Swagger UI must be whitelisted and accessible without authentication for developer visibility.

5. Future Enhancements (Post-MVP)

  • Role-Based Access Control (RBAC) (e.g., HR vs. Admin roles).
  • Pagination and sorting for the Employee Data Table.
  • Soft-deletion for historical records.
  • Employee profiles including contact information and hire dates.
