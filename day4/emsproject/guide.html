🚀 Full-Stack CRUD Application Setup Guide

Welcome to your newly generated Employee Management System. This guide will walk you through cloning, configuring, and running this project using VSCode Antigravity.

1. Workspace Structure

Your repository is split into two main sections:

  • fsdapp/ Backend - Spring Boot 3 Java Application.
  • frontend/ Frontend - React Vite Application.

2. Cloning and Opening in VSCode Antigravity

1. Open your terminal in VSCode.

2. Clone your repository (if hosted on Git):

git clone <your-repo-url>
cd emsproject

3. Ensure you open the root folder in VSCode so Antigravity has access to both workspaces.

3. Running the Spring Boot Backend

Database Configuration

The backend requires a MySQL database. Ensure the company database exists and verify the credentials in:

fsdapp/src/main/resources/application.properties

Start the Server

Open a terminal in VSCode, navigate to the backend directory, and run the Maven wrapper:

cd fsdapp
./mvnw spring-boot:run

The backend will start on http://localhost:8081. Swagger UI is available at http://localhost:8081/swagger-ui/index.html.

4. Running the React Frontend

Install Dependencies

Open a new split terminal in VSCode, navigate to the frontend directory, and install the Node packages:

cd frontend
npm install

Start the Dev Server

Run the Vite development server:

npm run dev

The frontend will be accessible at http://localhost:5173.

5. Authentication Setup

The application is secured with JWT Tokens.

We have pre-configured a default admin user in the remote database. You can log into the frontend immediately with:

  • Username: admin
  • Password: admin@123#

Alternatively, click "Register" on the login page to create a new user.

6. API Documentation (Swagger)

The backend includes auto-generated interactive OpenAPI documentation using the springdoc-openapi-starter-webmvc-ui library.

How it works:

  • The library automatically scans all @RestController endpoints to generate the schema.
  • The Swagger paths are explicitly whitelisted in SecurityConfig.java so you can view the docs without needing a JWT token.

Access the Docs:

While the Spring Boot server is running, visit:

  • Interactive UI: http://localhost:8081/swagger-ui/index.html
  • Raw JSON Spec: http://localhost:8081/v3/api-docs
