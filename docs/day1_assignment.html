Assignment

Spring Boot CRUD API

Employee Management System (No DB)

🎯 Objective

Create a simple REST API using Spring Boot to perform CRUD (Create, Read, Update, Delete) operations on employees without using any database.

Store all employee records in an ArrayList.

Step 1

Employee Class Setup

Create an Employee class with the following fields:

Employee.javaJava
private int id;
private String name;
private String department;
private int salary;

The class should include:

  • Parameterized Constructor
  • Getters
  • Setters
Step 2

Create a REST Controller

Create a class named EmployeeControllerCRUD.

Inside the controller, maintain an in-memory list:

EmployeeControllerCRUD.javaJava
private ArrayList employees = new ArrayList<>();
Step 3

API Requirements

1. Get All Employees

GET /employees

Returns the complete list of employees.

2. Add Employee

POST /employees

Accept employee details in JSON format. Example Request Body:

{
    "id": 1,
    "name": "John Doe",
    "department": "Engineering",
    "salary": 100000
}

3. Update Employee

PUT /employees/{id}

Search the employee using the path variable id. If found, update Name, Department, and Salary.

4. Delete Employee

DELETE /employees/{id}

Delete the employee whose id matches the path variable.

✅ Expected Output & Constraints

Outputs: Employees can be added, viewed, updated, and deleted via Postman.
Constraints: Do not use any database (MySQL, PostgreSQL, etc.). Do not use JPA or Hibernate.
Allowed: Spring Boot, REST Controller, ArrayList, Java Collections.
Bonus (Optional)

Additional APIs

  • 1. Get Employee by ID: GET /employees/{id}
  • 2. Search by Department: GET /employees/department/{department}
  • 3. Count Total Employees: GET /employees/count
  • 4. Highest Paid Employee: GET /employees/highest-salary
  • 5. Average Salary: GET /employees/average-salary

Submission: Submit the Spring Boot project zip, Employee classes, and Screenshots from Postman for all 4 requests.
